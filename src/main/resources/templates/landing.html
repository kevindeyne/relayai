AI-assisted workflows for developers and shareholders

Bring your team into a new era of productivity with RelayAI

Along with learning the expertise of team members and assigning relevant tasks to the right people, RelayAI takes into account everyone's workload to plan tasks, easily tracks time spent on projects, and simplifies things like: sprint planning, standup meetings, meeting SLA standards, and following up on replies.

Preview our alpha version

RelayAI is all about cutting down on time you spend managing your project, for you, your developers and your shareholders.

By applying smart defaults and adaptive learning we can handle the delegation and follow-up of tasks.

Learns developer expertise and workload

RelayAI understands your tasks, your workload and generates a tag cloud of expertise per user.

It can then match any new issues with the existing knowledge and direct tasks to the right people in your team.

However, users can always override this themselves. The workload will automatically adjust. It's all about smart defaults.

Integrates time-tracking

By guiding the user to track issues where applicable, we can provide accurate and automatic integrated time tracking of any issues being worked on.

This integrates with the sprint system to provide the developers with an accurate estimate on their sprint progress, and tools to ask colleagues for help.

This allows us also to make work in progress visible for teamleaders and shareholders.

Clarity for shareholders

Shareholders do not get to see the in depth tool the developers need, instead they get a cleanly formatted site customised just for them.

This helps them gain a better understanding of the project as a whole, which features are upcoming and if their input is required.

RelayAI will also follow-up and contact shareholders if time-sensitive feedback is necessary.

Technical details


Primary technologies used are Kotlin, Java 8, SASS, JOOQ and jQuery. This project runs as a self-contained JAR with Spring Boot on Amazon Web Services (AWS). Certification, hosting and load balancing is provided by AWS.

Monitoring is done internally with JavaMelody APM, and is available at the Monitoring station.

The entire project is open-source and available on GitHub.

Continious Integration for the project is based of the main trunk of the github branch, and automatically builds and runs the full test-suite on every deploy. This is provided by Travis CI.

Code review and static code analysis is performed through the Codacy project, which currently rates the project with a grade A project certification. Details can be found at Codacy.
